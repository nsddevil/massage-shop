---
description: 기능 구현 표준 프로세스 (Feature Dev Standard)
---

1.요구사항 분석 및 계획 수립 (Planning)

- 요청받은 기능을 분석하고, 수정이 필요한 파일 목록과 로직을 Artifacts로 작성하여 승인받는다.

  2.타입 정의 및 인터페이스 설계 (Type First)

- 코드를 짜기 전, 필요한 타입을 `types/` 폴더에 모듈별로 생성하고 `index.ts`에 export 설정을 마친다.
- any 는 사용하지 않는다.

  3.핵심 비즈니스 로직 개발 (Logic)

- 승인된 계획에 따라 함수를 작게 분리하여 로직을 구현한다. (룰의 'Clean Code' 준수)

  4.결과 보고 (Reporting)

- 모든 작업이 완료되면 변경 사항을 요약하여 한국어로 보고한다. (룰의 '한국어 소통' 준수)
